# 7 智能体公司内部通信 SOP

## 一、通信架构（当前已落地）
- 统一入口：飞书群 `GROUP_ID（安装时自动注入）`
- 统一调度：`rd-company`（研发总监）
- 角色分工：`role-product`、`role-tech-director`、`role-senior-dev`、`role-code-reviewer`、`role-qa-test`、`role-growth`
- 可视化看板：`${OPENCLAW_STATE_DIR}/workspace/rd-dashboard/index.html`

## 二、日常通信节奏
- 09:30 工作日晨会（可定时，当前关闭）
- 13:30 午间同步（可定时，当前关闭）
- 20:30 晚间复盘（可定时，当前关闭）
- 周一 10:00 周计划（可定时，当前关闭）

## 三、群内触发口令（建议固定用法）
- `#晨会`：生成今天 7 角色任务 + 风险 + 截止
- `#同步`：查看当前完成/进行中/阻塞
- `#阻塞 <问题>`：自动进入升级链路（含责任人与决策时限）
- `#决策 <事项>`：输出方案 A/B、推荐结论、执行人
- `#会审 <需求>`：触发跨角色会审（产品/技术/开发/review/测试）
- `#员工`：固定员工清单
- `#技能 程序员`：高级程序员固定技能矩阵
- `#github <需求>`：输出 issue/pr 拆解和推进计划
- `#qa <模块>`：输出测试范围、缺陷与发布建议

## 四、阻塞升级链路
1. 角色提交阻塞（问题、影响、责任人、期望决策时间）
2. 产品/技术总监/高级程序员给 2 个可行方案
3. 代码 Reviewer 给风险与审查意见
4. 测试工程师执行验证并给发布结论
5. 研发总监拍板（含截止时间）
6. 增长运营验证业务指标变化并做上线复盘

## 五、统一消息格式
- `来源角色`：产品/技术总监/高级程序员/代码Reviewer/测试工程师/增长运营
- `类型`：需求/技术方案/阻塞/决策/复盘
- `内容`：一句话结论 + 关键细节
- `负责人`：谁执行
- `截止时间`：具体到日期时间
- `状态`：todo/doing/done/blocked

## 六、看板刷新
```bash
cd "${OPENCLAW_STATE_DIR}/workspace/rd-dashboard"
./refresh.sh
```

打开 `http://127.0.0.1:8788` 查看角色日程、进度、cron 状态。
